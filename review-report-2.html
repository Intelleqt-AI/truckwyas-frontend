🔴 Runtime Crash — Fix Now
Bug · Crash
modelStats.synthetic_count is undefined → TypeError crash
src/pages/NewQuote.tsx · line 2311

Frontend calls .toLocaleString() on modelStats.synthetic_count, but the backend /api/v1/quotes/model-stats/ never returns that key — it returns real_quotes_count and training_sample_count. The TypeScript interface declares it but the value is always undefined at runtime.

This causes a TypeError: Cannot read properties of undefined crash whenever the AI suggestion panel renders.

FixEither add synthetic_count to the backend model-stats response, or replace the frontend reference with modelStats.training_sample_count and guard: {(modelStats?.synthetic_count ?? modelStats?.training_sample_count ?? 0).toLocaleString()}

QuoteWinProbabilityView: cross-tenant Customer lookup
core/views_ai_quote.py · line 1024

Customer.objects.get(id=client_id) has no company=request.user.company filter. An authenticated user can probe any customer ID across tenants. While no customer data is returned in the response directly, the foreign customer's quote history is used to compute the historical_acceptance_rate that informs the win probability prediction.

FixCustomer.objects.get(id=client_id, company=request.user.company) — raises DoesNotExist if cross-tenant.
Medium
InviteView.get() returns raw invite cache tokens to admins
core/views.py · line 2556–2568

The pending invites list response includes 'token': cache.get(f'invite_user_{u.id}') — the live invite token. Any ADMIN can retrieve valid tokens for pending users and replay the invite accept flow without the user's knowledge. Tokens in API responses end up in request logs.

FixRemove the 'token' field from the list response. Return 'has_pending_invite': True instead. Generate a fresh token only in the resend action.
